Build 2026060701 (1.3.1)

更新内容
• 长时间空闲修复：连接数小时无流量后，隧道可能显示"已连接"
  但实际不再转发任何流量（卡死的中继 → TCP 状态泄漏 → 内存
  耗尽 → 数据通路死锁）。现在空闲连接 5 分钟后自动回收，
  数据通路在内存压力下不再卡死。
• 网络切换：同一网络丢失或获得 IPv6（如 Wi-Fi 路由器关闭
  IPv6）现在能被检测到并自动重启数据通路——此前旧连接会
  一直黑洞直到超时。
• UDP 可靠性：代理服务器地址解析支持地址族回退，纯 IPv4
  网络下即使 DNS 先返回 IPv6 地址，UDP 转发（QUIC）也能
  正常工作。
• 安全性：扩展的内部控制 API 从固定的无鉴权端口改为随机
  端口 + 每次安装独立的密钥。
• 设置：移除了无实际作用的 IPv6 开关。
• meow-rs 引擎更新至 f48dac74：限制 gRPC 帧缓冲，
  fallback/url-test 代理组定期健康检查。

测试要点
• 整夜空闲：保持 VPN 连接过夜，早晨流量应立即恢复，无需
  重新连接。
• IPv6 切换：在双栈 Wi-Fi 上关闭路由器的 IPv6（或切换到
  纯 IPv4 热点），浏览应在几秒内恢复。
• QUIC 检查：纯 IPv4 网络开启 VPN 后，YouTube / Instagram
  视频应能流畅播放，无长时间卡顿。
• 问题反馈：https://github.com/madeye/meow-ios/issues
